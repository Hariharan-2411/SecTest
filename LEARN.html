0 · First-time setup

Do this once before anything else.

1

Build & load the extension: npm install && npm run build, then in chrome://extensions enable Developer mode → Load unpacked → pick the build/ folder.

2

Open the extension → Settings tab → add localhost and 127.0.0.1 to the Host Allowlist. The extension refuses to act on hosts that aren't allow-listed.

3

Keep Dry Run Mode ON (green badge). In dry run everything is simulated and logged, nothing is actually written to the page.

4

Serve this page over http://localhost:8000 (not file://) so recon and scope features behave like a real target, then reload the page so the content script injects.

The badge on the toolbar icon shows DRY or LIVE and turns into a counter as you approach the 20-actions/minute rate limit.

1 · Scan tab safe

Enumerate every form field on the current page and pick which ones to test.

1

Click Scan. The extension lists every input, textarea, select, and file input with its name/id/placeholder/type and a stable unique id.

2

Tick the checkbox on the fields you want to target (or select all). Text-like fields are eligible; date/number/range/color/file/select are skipped automatically for safety.

3

Use Insert Marker to drop a unique token like [TEST_1697…] into a field — the quickest way to confirm the extension can reach that input.

4

Modified fields get a colored outline on the page so you can see exactly what was touched. Nothing is ever auto-submitted.

Re-scan whenever the page changes (SPA navigation, new modal). Field references are tied to the scan session; a stale reference gives an "element not found" error.

Practice against the input-type zoo to watch which types get filled and which get skipped.

2 · Payloads tab

Choose a vulnerability family, pick a payload source, and run it across the fields you selected on Scan.

1

Pick a vulnerability family: XSS, SQL Injection, Command Injection, Path Traversal, SSRF, XPath, or LDAP.

2

Pick a payload source: Library (curated benign examples), File (upload a .txt, one payload per line), Text (type your own, one per line), or AI (generate — see the AI tab).

3

Click Run Test. Each selected field is filled with the payload(s), outlined, and every action is written to the audit log. No form is submitted — you review results and submit manually if you choose.

Payload validation blocks obviously dangerous strings (<script>, event handlers, DROP/DELETE/UNION, ../, eval()) on normal hosts. Sanctioned lab hosts in your allowlist bypass this so you can test real payloads.

3 · Recon tab

Understand the page and the target without attacking anything.

1

Page Recon → Overview / Forms / Storage: summarizes forms, links, and the names of storage keys/cookies (values are not read). Export Source saves the page HTML.

2

JS Change Monitor → Probe: fingerprints inline scripts so you can spot when a target's front-end changes between visits.

3

Active Recon and Fetch Recon Files reach out through the Companion Agent (next section) to pull subdomains, live hosts, ports, and nuclei findings.

4

Scheduled Watches: register a target to re-run recon on an interval and diff results, so new subdomains/endpoints surface automatically.

4 · Companion Agent active tools

The browser can't run nmap/nuclei. A tiny local HTTP service does, and re-checks scope on every request.

1

Start it: cd agent && cp .env.example .env, set AGENT_TOKEN to a long random string, then docker compose up --build. It listens on http://127.0.0.1:8787 (loopback only).

2

In the extension → Settings/Recon → Companion Agent: paste the Agent URL and token, click Check Health (shows installed tools), then Sync scope to push your program scope to the agent.

3

Run tools from Recon → Companion Agent:

ToolRiskPurpose
subfindersafeSubdomain enumeration
dnsxsafeDNS resolution
httpxsafeLive-host probing, titles, tech
naabuactivePort scan (top ports)
nmapactivePort/service detail
nucleiactiveKnown-vuln template scan

Profiles: quick, top1000, services (nmap). Active tools are gated by scope and the AGENT_ALLOW_ACTIVE switch.

Authorization first. Active tools touch the target. Only ever point the agent at assets you are explicitly authorized to test. Out-of-scope always wins — the agent returns 403 out_of_scope for anything not in scope.

5 · Checklist tab

A structured testing checklist so you cover a target methodically instead of ad-hoc.

1

Walk each item as you test the page — enumeration, auth, injection, access control, etc. Check items off to track coverage for the current target.

2

Pair it with the audit log (Settings → Export) to produce evidence of what you did and when.

6 · Programs tab

Track the bug-bounty programs you're working — scope and payouts.

1

Programs & Payouts stores each program's in-scope / out-of-scope assets. This scope is what Sync scope pushes to the Companion Agent, and what the extension checks before acting.

2

Keep scope accurate here first — every other feature (scan, recon, agent) trusts it.

7 · AI tab

Generate context-aware payloads with an LLM. Two backends are supported.

1

Groq (cloud) or Ollama (local). For Ollama: ollama pull llama3, export the origins env var so the extension is allowed, then ollama serve.

2

Set the base URL / model in the AI panel, click Test Connection, then generate. Generated payloads flow into the Payloads tab's "AI" source.

Ollama origins fix (403 errors): export OLLAMA_ORIGINS='["chrome-extension://*","http://127.0.0.1","http://localhost"]' before ollama serve. Default URL http://127.0.0.1:11434, model llama3.

8 · History tab

Every payload you run is saved to Payload History so you can re-run the ones that worked and build your own library over time.

1

Review past payloads, re-apply a useful one, or promote it into your Library/Text source for the next target.

9 · Settings tab

All the guardrails live here.

ControlWhat it does
Host AllowlistPer-host on/off; wildcard * supported. The extension only acts on allow-listed hosts.
Dry Run ModeON = simulate + log only. OFF = LIVE, every action needs explicit confirmation.
Audit LogTimestamp, URL, action, element, result for the last 100 actions. Export Log → JSON for evidence.
Rate limit20 actions/minute, throttled automatically to avoid accidental DoS.
Companion AgentAgent URL + token, health check, scope sync.
Turning off Dry Run means real values get written to real fields. Never do this on a target you aren't authorized to test. Nothing is auto-submitted, but LIVE writes are real.
