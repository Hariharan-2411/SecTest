# FILES.md â€” what every file does

A file-by-file map of the extension. Grouped by area; each entry is one line of
"what it's for." The design principle throughout: **pure logic lives in
`src/utils/` (no `chrome.*`, no network â€” unit-tested), and thin adapters in
`src/pages/` wire it to the browser.** So most of the intelligence is testable
offline, and the service worker / popup / content script are glue.

Data flows one way: sources (passive recon, active probes, the companion agent)
produce a normalized **Finding** â†’ the **validation gate** scores confidence â†’
**enrichment** adds CWE/CVSS â†’ **chains/report builder** shape drafts â†’ a human
validates and submits. Nothing auto-submits.

## Companion agent (`agent/`)

A zero-dependency, **loopback-only** HTTP service that runs allow-listed terminal
recon tools (the browser can't) against **in-scope** targets and returns results.

| File | Purpose |
|---|---|
| `server.js` | The HTTP service: `/health`, `/scan`, `/run`, `/scope`, `/watches`; re-checks scope server-side on every request; token-auth, rate-limited, `127.0.0.1`-bound. |
| `lib/tools.js` | Tool registry (`subfinder`/`httpx`/`nmap`/`nuclei`/â€¦) with risk tags + argument builders; runs via `execFile` (no shell). |
| `lib/scope.js` | Server-side scope model (mirrors the extension's `scope.js`). |
| `lib/parse.js` | Normalizes each tool's raw stdout into a compact structured shape. |
| `lib/oob.js` | Out-of-band interaction collector â€” confirms blind vulnerabilities. |
| `lib/recondiff.js` | Diffs two recon snapshots so scheduled watches alert on new attack surface only. |
| `lib/watches.js` | Scheduled-recon "watch" model (a recurring job against one in-scope target). |
| `lib/store.js` | Tiny JSON persistence so watches survive agent restarts. |
| `lib/notify.js` | Webhook formatting for the agent's own scheduler alerts. |
| `Dockerfile` / `docker-compose.yml` | Containerized run (`docker compose up` â†’ `http://127.0.0.1:8787`). |
| `package.json` | Agent metadata (no runtime deps). |
